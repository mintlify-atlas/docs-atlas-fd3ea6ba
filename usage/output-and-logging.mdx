---
title: "Output and logging"
description: "Control WPX terminal output verbosity and save findings to a plain-text file using the --quiet and --output flags."
---

By default, WPX prints a banner, real-time status messages, progress bars, and all findings to the terminal with ANSI color codes. Two flags let you tighten up what gets printed and persist results to disk.

## Flags

**`-q / --quiet`** — suppresses the banner, status lines (prefixed `[*]`), and progress bars. Only findings (`[+]`) and warnings (`[!]`) are printed.

**`-o / --output FILE`** — writes the full output to a plain-text file. ANSI color codes are stripped so the file is clean for storage, diffing, or piping into other tools.

These flags are independent — you can use either or both together.

## Examples

### Quiet mode

Run a scan with minimal terminal noise:

```bash
python3 wpx.py -u https://example.com -q
```

### Save findings to a file

Write all output to `results.txt` while still displaying it in the terminal:

```bash
python3 wpx.py -u https://example.com --output results.txt
```

### Silent scan saving to file

Suppress terminal output and write clean findings to a file — useful in CI pipelines or scheduled scans:

```bash
python3 wpx.py -u https://example.com --quiet --output results.txt
```

With Docker, mount a host directory to write the file outside the container:

```bash
docker run -v $(pwd):/output ghcr.io/greg-randall/wpx -u https://example.com --quiet -o /output/results.txt
```

## Output format

WPX prefixes each line to indicate its type:

| Prefix | Meaning |
|--------|---------|
| `[+]`  | Finding — a detected plugin, user, header, or vulnerability |
| `[*]`  | Status — scan progress and informational steps |
| `[!]`  | Warning — potential issue or non-fatal error |
| `[i]`  | Info — supplementary detail (e.g., count of discovered items) |

`--quiet` suppresses `[*]` and `[i]` lines, leaving only `[+]` and `[!]`.
