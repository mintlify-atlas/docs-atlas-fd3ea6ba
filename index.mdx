WPX (WordPress X-Ray) is an open-source security scanner built for authorized WordPress penetration testing. It uses Camoufox — a high-fidelity headless browser — to solve Cloudflare and WAF challenges, then mirrors the session to perform fast, asynchronous enumeration of plugins, themes, users, and vulnerabilities.

## What WPX does

WPX automates the reconnaissance phase of a WordPress security assessment. A single command launches a multi-stage scan:

<Steps>
  <Step title="WAF bypass">
    Camoufox opens a real browser, solves any JS challenge or CAPTCHA, and extracts the session cookies and User-Agent. The mirrored `curl_cffi` session then inherits these tokens so every subsequent request looks indistinguishable from the browser that solved the challenge.
  </Step>
  <Step title="Passive discovery">
    WPX fetches the homepage and a known-404 page, then parses HTML for plugin paths, theme slugs, and author links — with no additional requests.
  </Step>
  <Step title="Active enumeration">
    Plugin slugs from a ranked list of up to 110,000 plugins are brute-forced concurrently. WPX checks HTTP 200 and 403 responses to confirm presence, then fetches readme files and asset query strings to detect versions.
  </Step>
  <Step title="User enumeration">
    Five independent techniques — REST API, author archive probing, oEmbed, RSS feed, and passive HTML — are tried in sequence. Blocked methods are recorded as security improvements.
  </Step>
  <Step title="Vulnerability lookup">
    If you provide a WPScan API key, WPX queries the WPScan Vulnerability Database for every detected plugin and filters results to only the CVEs that apply to the installed version.
  </Step>
</Steps>
