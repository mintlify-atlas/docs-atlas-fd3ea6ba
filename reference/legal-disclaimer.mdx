## Authorized use only

WPX is a security tool intended exclusively for authorized security testing. You may only use WPX against websites and systems for which you have explicit, written permission from the owner or operator.

**Authorized** means you have one of the following:

- Written consent from the site owner or the organization responsible for the target
- A signed scope-of-work or penetration testing agreement that explicitly includes the target domain
- You own the site yourself

Running WPX against a target without this authorization — regardless of intent — is not a permitted use of this tool.

## Responsible disclosure

If you discover vulnerabilities during an authorized test using WPX, follow responsible disclosure practices:

1. **Do not exploit** vulnerabilities beyond what is necessary to confirm they exist and falls within your agreed scope.
2. **Report to the site owner** directly, or through a coordinated disclosure process if one exists.
3. **Allow reasonable time** for the issue to be remediated before publishing details publicly.
4. **Follow your agreement** — if you have a contract or scope document, it may specify disclosure timelines and procedures.
