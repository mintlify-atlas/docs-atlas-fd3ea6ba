This page walks you through running your first WPX scan and understanding the output. Before you begin, make sure you have [installed WPX and fetched the Camoufox browser binary](/installation).

<Warning>
  Only scan WordPress sites you own or have explicit written permission to test. Unauthorized scanning is illegal.
</Warning>

## Run a basic scan

<Steps>
  <Step title="Launch the scanner">
    Pass the target URL with the `-u` flag. WPX launches Camoufox, solves any WAF challenge, mirrors the session, and begins enumeration automatically.

    <Tabs>
      <Tab title="pip install">
        ```bash
        python3 wpx.py -u https://example.com
        ```
      </Tab>
      <Tab title="Docker">
        ```bash
        docker run ghcr.io/greg-randall/wpx -u https://example.com
        ```
      </Tab>
    </Tabs>
  </Step>

## Run a vulnerability scan

To look up CVEs for detected plugins, pass a [WPScan API key](https://wpscan.com/api) with `--api-key`. WPX queries the WPScan Vulnerability Database and filters results to only the CVEs that apply to the installed version.

```bash
python3 wpx.py -u https://example.com --api-key YOUR_KEY
```

Plugins with active vulnerabilities are flagged `[VULNERABLE]` in the results, with the CVE title, fixed-in version, and reference URLs.

## Save results to a file

Use `-o` to write plain-text output (no ANSI color codes) to a file, and `-q` to suppress the banner and progress lines:

```bash
python3 wpx.py -u https://example.com --quiet --output results.txt
```
